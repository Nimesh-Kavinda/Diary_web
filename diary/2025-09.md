# Developer Diary — September 2025

---

## 2025-09-08

- Kicked off the **gLabs Automator** project with the very first commit. Set up the repository structure, installed dependencies, and committed the initial Vite + React scaffolding. This was the starting gun for a new wave of project work.

---

## 2025-09-09

- Spent the full day building out the gLabs Automator landing page. Built the base page structure, a detailed footer, and a smooth scroll navigation button. Added the Product Hunt badge to the site, resolved a dark mode button highlight bug, and corrected several font colour inconsistencies. Created the Privacy Policy pages and fixed a Tailwind CDN loading issue that was causing styles to not apply on first load.

---

## 2025-09-10

- **gLabs Automator**: Deployed the completed landing page to Firebase Hosting. Hit a frustrating 404 page override issue where the custom 404 page was being served for all routes — fixed by adjusting the `firebase.json` rewrites. Addressed navigation and responsive layout bugs. Compressed images for lazy loading to improve initial page speed.
- **busy_books**: Started the busy_books project. Completed three major landing page sections in the initial commit — this was a satisfying first day for a project that would consume the next six weeks.

---

## 2025-09-11

- Productive day building the busy_books landing page UI. Added a mobile mockup component and built the **BookSlider** — a horizontally scrollable row of book covers. Created the Desktop Mockups section showing the app on screen, and the Mobile Mockup section. Built out the Summary Audio section (a teaser audio player on the landing page) and added a scroll-down button. Fixed several console warning issues that were cluttering the browser devtools.

---

## 2025-09-12

- Focused on polish work. Fixed icon rendering issues throughout the landing page components. Attempted to resolve a batch of React console warnings related to missing keys and prop validation — partially resolved them but some required deeper investigation into component structure.

## 2025-09-14

- Worked through the weekend with good progress. Changed the page title and favicon to match the busy_books brand. Redesigned the scrolling navigation UI that was looking too basic. Fixed dark mode colour inconsistencies that stood out during a full-page review. Built the **Footer** and **Download Section** (directing users to app stores), then built the **Testimonials** section with user quotes and avatars.

## 2025-09-15

- Reviewed and tested all the landing page components from the weekend's work. Planned the inner app structure: the onboarding flow, login/signup screens, and dashboard architecture. Drafted the routing strategy for separating authenticated users from guest access.

---

## 2025-09-16

- Made major structural changes. Reorganised the entire folder structure for better scalability — separating components, pages, hooks, and services into their own directories. Fixed lingering gradient display issues in the hero section. Built the **Login page**, the full **Onboarding page** (a multi-step genre selection flow), and the **SignUp page**.

---

## 2025-09-17

- Added logo icons to the assets folder and made background colours dynamic across the app using a theme context. Made book cover images dynamic by replacing hardcoded `src` values with props. Built the example **Dashboard page** and the **Explore page** (browsing categories and popular books). Fixed an image rendering issue in the BookCard component where images were overflowing their containers.

---

## 2025-09-18

- The most feature-dense day of the project so far. Built nearly every inner application page:
  - Added the mobile logo to the side navigation bar.
  - Aligned components on the Settings page and changed the entire site font.
  - Updated the landing page navigation and changed the header and sidebar background colours.
  - Refactored the assets folder structure; replaced the nav settings link with a sign-out button in the header.
  - Fixed image border issues; updated the UserBanner component styling.
  - Built all **Info pages** and their child components, the **Library page**, the **Rate Plan page**, and the **Settings page**.

---

## 2025-09-19

- Finalised most of the app's page structure. Aligned padding and margins in the Book Details and Listen pages. Made the page header dynamic and updated header colours. Improved the **Audio Player** component behaviour and the BookInfo pages. Adjusted text colours throughout for readability. Built the **Book Listing page**, the **Book Read page**, the **Contact page**, and the **Single Book detail page**. The app now had every page built, if not yet wired to live data.

## 2025-09-22

- Switched to the **HabbitTracker_V2** project — the second major project of this period. Initialized the project from scratch with an entirely new architecture. Spent the day doing a comprehensive code quality pass across all initial components: refactored the Analytics component for readability, and cleaned up code formatting in Dialog, Select, HabitCard, Login, ForgotPassword, Profile, Reorder, and SignUp components — standardising quote style, spacing, and prop naming conventions.

---

## 2025-09-23

- Back to **busy_books** for authentication integration. Implemented route validation functions that distinguish logged-in users from guest users — using React Router's `<Navigate>` element to redirect guests away from protected pages. Made the user's display name and email visible in the profile section. Built the full **Login**, **Registration**, **Sign In**, and **Sign Out** flows using Firebase Auth (`signInWithEmailAndPassword`, `createUserWithEmailAndPassword`).

---

## 2025-09-24

- Deep Firestore integration work. Added a logo to the app's loading state. Implemented **category fetching from Firestore** in the Explore component — books are now grouped by real categories from the database. Refactored the BookCard and CategorySection components for consistency in prop handling and render logic. Multiple rounds of Explore component refactoring to improve loading state management and author data structures.

---

## 2025-09-25

- Major backend pivot. Added **Axios** as the HTTP client and switched the Explore component to fetch books from the new Express API instead of directly from Firestore. Created the `backend/` folder with the Express + Node.js server scaffolding. Fixed a data inconsistency bug between the popular books and latest books endpoints where the sort order was reversed. Refactored all category book fetching to use dedicated API endpoints.

---

## 2025-09-26

- API buildout day. Added **author routes** and a controller to fetch popular authors; updated the Explore component to retrieve authors from the API. Implemented **book category routes**, a category controller, and the Mongoose category model. Enhanced all book fetching methods to return only selected fields, reducing response payload size. Integrated MUI (Material UI) icons and enhanced the CategoryBadge component with an icon prop. Removed fallback image logic from AuthorCard. Multiple refactors to tighten up route structure and fetch consistency.

---

## 2025-09-27

- Saturday work on the Author model. Added the `bio` field to the author document when fetching popular authors — it was previously being omitted. Set a default placeholder `photoUrl` in the Author model to prevent broken image displays when an author has no photo.

---

## 2025-09-28

- Sunday feature work. Implemented the **'For You' books feature**: built the personalised book fetching logic using the user's onboarding topic preferences, integrated it into the UserDashboard. Enhanced the CategoryBadge button with a box shadow effect and subtle reduced-opacity inactive state.

---

## 2025-09-29

- Built out all remaining personalisation endpoints in a productive single day. Added an endpoint to fetch books filtered by user topic preferences. Implemented the **Read Books** feature (tracking which books a user has finished). Added **recently viewed books** tracking. Built the **Bookmarked Books** feature. Completed the full **'For You' categories** system — all backed by user authentication and Firebase user documents.

---

## 2025-09-30

- UI polish and infrastructure day. Added an endpoint to fetch a single book by ID with proper error handling. Built **loading skeletons** for BookCard, CategoryBadge, and all dashboard/explore loading states — dramatically improving the perceived performance. Added the 'See All Library Books' endpoint. Built the **'View All' page** with routing from the dashboard. Enhanced Library and ViewAll with category-based navigation. Multiple refactors to improve code readability and the rate limiter configuration.
